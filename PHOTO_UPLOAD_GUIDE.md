# Руководство по загрузке фото товаров

## ✅ Что реализовано

### 1. Хранение фото
- **Фото хранятся локально** в папке `media/products/YYYY/MM/DD/` (организовано по датам)
- **Пути к фото хранятся в БД** в таблице `ProductImage`
- Каждый товар может иметь **от 1 до 6 фото**

### 2. Способы загрузки фото

#### Способ 1: Django Admin Panel (Jazzmin) - `/ads/`
1. Перейдите в админку: `http://localhost:8000/ads/`
2. Войдите под своими учетными данными
3. Выберите товар или создайте новый
4. В разделе "Изображения товара" внизу страницы:
   - Нажмите "Добавить изображение товара"
   - Загрузите фото (можно до 6 штук)
   - Установите порядок (поле "Порядок": 0, 1, 2, ...)
5. Сохраните товар

#### Способ 2: Кастомная панель - `/admin/panel`
1. Перейдите: `http://localhost:8000/admin/panel`
2. Нажмите "+ Добавить товар"
3. В поле "Фото" выберите несколько файлов (Ctrl+клик или Shift+клик)
4. Минимум 1, максимум 6 фото
5. Первое фото = обложка товара
6. Заполните остальные поля и сохраните

## 📸 Автоматическое переключение фото

### На десктопе (PC)
- **При наведении мыши** на карточку товара фото автоматически переключаются
- Интервал: 800ms
- При уходе мыши возвращается первое фото

### На мобильных
- **Автоматическое переключение** для товаров, которые видны на экране
- Интервал: 2 секунды
- Используется IntersectionObserver для определения видимости

### В модальном окне
- Кнопки навигации `‹` и `›` для переключения
- Миниатюры всех фото внизу
- Клик по миниатюре = переход к этому фото

## 🗂️ Структура БД

### Таблица `ProductsApp_product`
- `id` - ID товара
- `name` - название
- `category` - категория
- `price` - цена
- `discount_percent` - скидка в %
- `badge` - метка (new/hot)
- `in_stock` - в наличии
- `_sizes` - размеры (JSON)
- `description` - описание
- `created_at` - дата создания
- `updated_at` - дата обновления

### Таблица `ProductsApp_productimage` (новая!)
- `id` - ID изображения
- `product_id` - FK к товару
- `image` - путь к файлу (например: `products/2026/06/27/photo.jpg`)
- `order` - порядок отображения (0, 1, 2, ...)

## 🔧 Технические детали

### Установленные зависимости
```bash
pip install Pillow  # Для работы с изображениями
```

### Настройки Django (config/settings.py)
```python
MEDIA_URL = '/media/'
MEDIA_ROOT = BASE_DIR / 'media'
```

### Валидация
- **Минимум 1 фото** при создании товара (обязательно!)
- **Максимум 6 фото** на товар
- При загрузке в админ-панели: валидация на стороне views.py
- При обновлении: можно не загружать новые фото (сохранятся старые)

### Удаление фото
При удалении записи `ProductImage` из БД:
- Автоматически удаляется файл с диска (метод `delete()` в models.py)
- При замене фото: старые удаляются, новые создаются

## 🚀 Как использовать

### 1. Запустить сервер
```bash
python manage.py runserver
```

### 2. Добавить товар с фото
- Админка: `http://localhost:8000/ads/`
- Или панель: `http://localhost:8000/admin/panel`

### 3. Просмотреть на сайте
- Главная: `http://localhost:8000/`
- Каталог: `http://localhost:8000/catalog/`

### 4. Проверить автопереключение
- **Десктоп**: наведите мышь на товар с несколькими фото
- **Мобильный**: откройте в браузере с шириной < 768px

## ⚠️ Важно

1. **Первая загрузка**: При создании товара ОБЯЗАТЕЛЬНО загрузите минимум 1 фото
2. **Редактирование**: Можно не загружать новые фото — останутся старые
3. **Замена**: Если загрузите новые фото при редактировании — старые удалятся
4. **Порядок**: Первое фото (order=0) = обложка товара
5. **Форматы**: PNG, JPG, JPEG, GIF, WebP — все поддерживаются (Pillow)

## 🐛 Возможные проблемы

### Проблема: "Cannot use ImageField because Pillow is not installed"
**Решение**: 
```bash
pip install Pillow
```

### Проблема: Фото не отображаются на сайте
**Решение**: Проверьте настройки в `config/urls.py`:
```python
if settings.DEBUG:
    urlpatterns += static(settings.MEDIA_URL, document_root=settings.MEDIA_ROOT)
```

### Проблема: Загружается только 1 фото вместо нескольких
**Решение**: Убедитесь, что в HTML форме стоит `multiple`:
```html
<input type="file" name="images" accept="image/*" multiple>
```
